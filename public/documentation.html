myFlix Movies

What the API does:

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Business Logic: URL: HTTP Method: Request body data format Response body data format
Return a list of movies to the user /movies GET None An array of all movies
Return data about a single movie by title to the user /movies/:title GET None A JSON object with data about a single movie containting title, description, genre, director, image url, and whether its featured or not.
Return data about a movie genre by name/title /movies/:genre GET None A JSON object with a description of the movie genre.
Return data about a director by name /movies/:director GET None A JSON object with a description of the director.
Allow new users to register /users/ POST A JSON object with user data including id, name, and favoriteMovies. A JSON object with user info and computer generated id.
Allow users to update their user information users/:id PUT A JSON object with the user info to be changed; should have one or more of the following: {"id":" ", "name":" ", "favoriteMovies":[" "]} A JSON object with updated user info.
Allow users to add a movie to their list of favorites /users/:info/movies/:movieTitle Post None A text that the movie has been added to their array.
Allow users to remove a movie from their list of favorites /users/:info/movies/:movieTitle DELETE None A text confirming the movie has been removed from their array.
Allow existing users to deregister /users/:info DELETE None A text that confirms the user was deleted.
